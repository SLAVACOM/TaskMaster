# S3CloudeStorage — карта сервиса

## Обзор

| Параметр | Значение |
|---|---|
| Язык | Kotlin (JVM 21) |
| Порт | 8085 |
| База данных | нет (Redis для кэша) |
| Фреймворк | Spring Boot 4.0.5 |
| Lombok | да |

**Роль:** Генерирует presigned URL для загрузки и скачивания объектов из S3-совместимого хранилища. Кэширует сгенерированные URL в Redis, чтобы не создавать новую подпись при повторных запросах. JPA-сущностей нет.

---

## Конфигурация (переменные окружения)

| Переменная | Назначение |
|---|---|
| S3 endpoint | URL хранилища (MinIO / AWS S3) |
| S3 access key / secret | Credentials |
| S3 bucket name | Бакет |
| Redis host/port | Кэш presigned URL |

---

## REST API

### S3Controller

| Метод | URL | Описание |
|---|---|---|
| POST | `/s3/upload-url` | Получить presigned URL для загрузки одного файла |
| POST | `/s3/upload-urls` | Получить presigned URL для загрузки нескольких файлов |
| POST | `/s3/download-url` | Получить presigned URL для скачивания одного файла |
| POST | `/s3/download-urls` | Получить presigned URL для скачивания нескольких файлов |

> Все эндпоинты возвращают временные подписанные URL. Клиент напрямую загружает/скачивает файл в S3, минуя этот сервис.

## Точки расширения / что отсутствует

- [ ] **Интеграция с TaskService** — TaskService хранит ключи S3 в поле `files`, но не вызывает S3CloudeStorage для получения URL; клиент должен делать два отдельных запроса
- [ ] **Удаление файлов** — нет эндпоинта для удаления объектов из S3 (при удалении задачи файлы остаются в хранилище)
- [ ] **Листинг файлов** — нет эндпоинта для просмотра содержимого бакета/папки
- [ ] **Авторизация** — нет проверки, что запрашивающий пользователь имеет право на доступ к конкретному объекту
- [ ] **TTL конфигурация** — время жизни presigned URL и Redis-кэша должны быть синхронизированы
- [ ] **Мультибакетность** — сервис работает с одним бакетом; нет поддержки разных бакетов для разных типов данных
