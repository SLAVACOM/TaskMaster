# NotificationService — карта сервиса

## Обзор

| Параметр | Значение |
|---|---|
| Язык | Kotlin (JVM 21) |
| Порт | 8090 |
| База данных | PostgreSQL `event_db` |
| Фреймворк | Spring Boot 4.0.5 |
| Lombok | нет |

**Роль:** Отправляет уведомления по Email и Telegram. Принимает события из Kafka и HTTP-запросы. Использует таблицу идемпотентности (`ProcessedEvent`) для предотвращения дублирования при повторной доставке Kafka-сообщений.

---

## Доменные сущности

### `ProcessedEvent` (`event_db.processed_events`)

| Поле | Тип | Описание |
|---|---|---|
| `id` | UUID/String | PK (event ID из Kafka) |
| — | — | Хранит ID обработанных событий для идемпотентности |

> Минимальная сущность: только для дедупликации. Реальное содержимое — в Kafka-событии.

## Точки расширения / что отсутствует

- [ ] **Топик `notifications.events` не имеет продюсера** — в проекте нет сервиса, который публикует в этот топик (только `user-login` из Auth-service активно используется)
- [ ] **Push-уведомления** — только email + Telegram; нет Web Push, FCM, APNs
- [ ] **История уведомлений** — нет хранения отправленных уведомлений (невозможно просмотреть историю)
- [ ] **Шаблоны** — нет системы шаблонов для писем (жёстко закодированный текст)
- [ ] **Предпочтения пользователя** — нет настройки: какой канал использовать, какие типы уведомлений получать
- [ ] **Retry-механизм** — нет явной обработки ошибок отправки (SMTP недоступен, Telegram API лимит)
- [ ] **Интеграция с OrganizationService** — при принятии/отклонении приглашения не приходит событие
- [ ] **Интеграция с TaskService** — при назначении задачи/смене статуса не приходит событие
